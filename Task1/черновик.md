в витрину данных не нужно добавлять медицинские карты, истории болезней и результаты медицинских исследований. Эти данные компания не будет использовать для аналитики.

Интеграция новых бизнесов без дописывания огромного количества бизнес-логики в DWH.

необходимо:
  - отказаться от легаси-систем
  - мигрировать на новые сервис
  - предоставить весомое обоснование

структуре компании
    Головной офис.
    Клиники.
    ИИ-сервисы для работы с медицинскими данными в виде отдельной компании.
    Финтех-сервисы в виде отдельной компании с банковской лицензией.

продукты:
  - Финтех-сервисы.
  - Внутренняя медицинская система-DWH с клиентским интерфейсом и дополнительной бизнес-логикой под финтех- и ИИ-сервисы.
  - Много кастомизаций BI-системы поверх DWH.
  - ИИ-сервисы для работы с медицинскими данными.
  - Слой интеграций через старую шину данных.
